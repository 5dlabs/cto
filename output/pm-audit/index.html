Full OAuth lifecycle for 16 Linear agent apps: authorization_code flow, token refresh, client_credentials minting, and persistent storage in Kubernetes secrets.
stateful Linear OAuth Kubernetes
›

Routes

Background task running every 5 minutes. Checks all 16 agent tokens for expiry. Proactively refreshes or mints new tokens before they expire. Concurrency-limited to 3 simultaneous refreshes.
stateful Linear OAuth Kubernetes

Migration Decision Matrix

✅ Safe to Move to Morgan

Business logic that Morgan should own
  • Intake workflow routing (PRD → CodeRun)
  • Play workflow submission & cancellation
  • Agent interaction routing (@mentions, CI failures)
  • GitHub/GitLab event business logic (review triggers, merge handling)
  • Callback processing (intake-complete, play-complete, pr-created)
  • Document → ConfigMap syncing
  • Play state machine (agent → Linear state mapping)
  • Detection module (language/framework/agent selection)

🚫 Should Stay in PM

Infrastructure/platform concerns Morgan shouldn't own
  • HMAC-SHA256 webhook signature verification
  • Multi-agent OAuth token lifecycle (refresh, mint, store)
  • Background token health manager (5min proactive refresh)
  • Kubernetes secret management for tokens
  • OAuth authorization flow (start, callback)
  • Agent pod communication router (session cache, K8s pod discovery)
  • Session tracker (in-memory state, reverse lookups)
  • Health checks & readiness probes

⚠️ Shared / Boundary

Could go either way depending on approach
  • Linear GraphQL client (lib, used by both)
  • Activity emission (currently used by PM + agent pods)
  • Onboarding state machine (WIP, not yet wired)
  • Morgan dispatch module (becomes unnecessary if Morgan receives directly)

💡 Recommendation

Complete the existing Shadow → Morgan cutover
  • PM shrinks to: auth gateway + token broker + agent router
  • Morgan owns: all business logic decisions
  • Shadow mode lets you dark-launch safely
  • No breaking changes to external webhook URLs
  • OAuth/token management stays centralized
  • Linear client already exists as separate crate (linear-sink)
