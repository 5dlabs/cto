Implement subtask 10003: Enable and configure Kubernetes API server audit logging

## Objective
Configure the Kubernetes API server audit policy to log security-relevant events (authentication, authorization, secret access, RBAC changes) and ship audit logs to persistent storage.

## Steps
1. Create an audit policy YAML defining rules:
   - Log at `RequestResponse` level: secret read/write, RBAC resource changes, pod exec, token requests.
   - Log at `Metadata` level: all other write operations.
   - Log at `None` level: health checks, kube-system read-only operations (to reduce noise).
2. Configure the API server to use the audit policy (method depends on cluster provider: managed K8s may use provider-specific settings; self-managed requires `--audit-policy-file` and `--audit-log-path` flags).
3. Set audit log rotation parameters (`--audit-log-maxage=30`, `--audit-log-maxbackup=10`, `--audit-log-maxsize=100`).
4. If using a managed cluster, enable the provider's audit log feature and configure export to the cluster's logging pipeline.
5. Verify audit logs are being generated by performing a test action (e.g., reading a secret) and checking the log output.

## Validation
Perform a `kubectl get secret` and verify the action appears in audit logs with correct user, verb, and resource; perform a pod exec and verify it's logged at RequestResponse level; verify health check endpoints are NOT logged (noise reduction); confirm log rotation is active.
