Implement subtask 9004: Define and apply Kubernetes NetworkPolicies to restrict inter-service traffic

## Objective
Design an allowed communication matrix for all services and implement Kubernetes NetworkPolicies that default-deny all traffic and explicitly allow only required service-to-service, ingress, and egress paths.

## Steps
1) Document the full service communication matrix: which services talk to which (e.g., frontend -> API gateway, API gateway -> backend services, backend services -> PostgreSQL/Redis/Kafka, cloudflared -> Morgan service, Prometheus -> all services for scraping). 2) Create a default-deny-all ingress and egress NetworkPolicy for the namespace: `spec.podSelector: {}` with `policyTypes: [Ingress, Egress]` and empty ingress/egress arrays. 3) For each allowed communication path, create a targeted NetworkPolicy that selects the destination pods and allows traffic from the specific source pods using label selectors. 4) Allow DNS egress (port 53 UDP/TCP to kube-dns) for all pods. 5) Allow egress to external services (Cloudflare, external APIs) as needed by specific pods. 6) Label all pods consistently to support NetworkPolicy selectors (e.g., `app: morgan`, `app: api-gateway`). 7) Apply policies incrementally, testing each one to avoid locking out services.

## Validation
After applying default-deny, verify that unlabeled test pods cannot reach any service. For each allowed path, verify connectivity using `kubectl exec` + `curl` or `nc` between the source and destination pods. For each denied path, verify that connection attempts time out or are refused. Specifically test: frontend can reach API gateway, API gateway can reach backends, backends can reach their databases, Prometheus can scrape all services, and cloudflared can reach Morgan. Verify DNS resolution works from all pods.
