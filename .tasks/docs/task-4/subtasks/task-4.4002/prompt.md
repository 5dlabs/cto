Implement subtask 4002: Write database migrations for finance schema

## Objective
Create SQL migration files for all finance domain tables: invoices, invoice_line_items, payments, payroll_entries, currency_rates, and supporting tables.

## Steps
1. Use sqlx-cli migrations (sqlx migrate add). 2. Create migration for invoices table: id (UUID), project_id (nullable, for cross-service reference), customer_name, customer_email, status (enum: draft, sent, paid, overdue, cancelled, void), currency (VARCHAR(3), ISO 4217), subtotal (BIGINT, cents), tax_amount, total_amount, due_date, paid_date (nullable), stripe_invoice_id (nullable), notes, created_at, updated_at. 3. Create migration for invoice_line_items: id, invoice_id (FK), description, quantity (DECIMAL), unit_price (BIGINT, cents), line_total, created_at. 4. Create migration for payments: id, invoice_id (FK), amount (BIGINT, cents), currency, payment_method (enum: stripe, bank_transfer, cash, check), stripe_payment_intent_id (nullable), status (enum: pending, succeeded, failed, refunded), received_at, created_at. 5. Create migration for payroll_entries: id, crew_member_id, pay_period_start, pay_period_end, hours_worked (DECIMAL), hourly_rate (BIGINT, cents), gross_amount, deductions, net_amount, currency, status (enum: draft, approved, paid), paid_date (nullable), created_at. 6. Create migration for currency_rates: id, base_currency, target_currency, rate (DECIMAL(20,10)), fetched_at, created_at. Add unique constraint on (base_currency, target_currency, fetched_at). 7. Add indexes on invoice status, payment status, payroll status, and currency_rates lookup.

## Validation
sqlx migrate run succeeds against a clean PostgreSQL database; all tables, columns, constraints, and indexes exist; sqlx migrate revert rolls back cleanly; sqlx prepare generates offline query data successfully.
