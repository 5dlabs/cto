Implement subtask 4002: Define data models and SQLx migrations for Invoice, Payment, Payroll, and financial enums

## Objective
Create the PostgreSQL schema migrations and corresponding Rust struct models for all finance domain entities: Invoice, InvoiceLineItem, Payment, PayrollRecord, CurrencyRate, and supporting enums (InvoiceStatus, PaymentStatus, PaymentMethod, PayrollFrequency, Currency).

## Steps
1. Create `migrations/` directory and use sqlx-cli for migration management. 2. Migration 001: Create `finance` schema. 3. Migration 002: Create enum types — `invoice_status` (draft, sent, paid, overdue, cancelled), `payment_status` (pending, processing, completed, failed, refunded), `payment_method` (stripe, bank_transfer, cash, check), `payroll_frequency` (weekly, biweekly, monthly), `currency` (USD, EUR, GBP, CAD, AUD, etc.). 4. Migration 003: Create `finance.invoices` table — id (UUID PK), invoice_number (unique serial), client_id (UUID), project_id (UUID nullable), status (invoice_status), currency (currency), subtotal (NUMERIC(15,2)), tax_amount (NUMERIC(15,2)), total (NUMERIC(15,2)), due_date (DATE), issued_date (DATE), paid_date (DATE nullable), notes (TEXT nullable), created_at, updated_at. 5. Migration 004: Create `finance.invoice_line_items` — id, invoice_id (FK), description, quantity (NUMERIC), unit_price (NUMERIC(15,2)), amount (NUMERIC(15,2)). 6. Migration 005: Create `finance.payments` — id (UUID PK), invoice_id (FK nullable), amount (NUMERIC(15,2)), currency, status (payment_status), method (payment_method), stripe_payment_intent_id (VARCHAR nullable unique), stripe_charge_id (VARCHAR nullable), reference_number (VARCHAR nullable), paid_at (TIMESTAMP nullable), created_at, updated_at. 7. Migration 006: Create `finance.payroll_records` — id, employee_id (UUID), period_start (DATE), period_end (DATE), gross_amount (NUMERIC(15,2)), deductions (NUMERIC(15,2)), net_amount (NUMERIC(15,2)), currency, frequency (payroll_frequency), status (payment_status), paid_at (TIMESTAMP nullable), created_at, updated_at. 8. Migration 007: Create `finance.currency_rates` — id, base_currency, target_currency, rate (NUMERIC(18,8)), fetched_at (TIMESTAMP), created_at. Add unique constraint on (base_currency, target_currency). 9. In `src/models/`, create Rust structs with sqlx::FromRow derives matching each table. Create enum types with sqlx::Type derives. Add serde Serialize/Deserialize on all structs. 10. Create request/response DTOs: CreateInvoiceRequest, UpdateInvoiceRequest, CreatePaymentRequest, CreatePayrollRequest, etc.

## Validation
All migrations run successfully against a clean PostgreSQL database. Rust structs compile and can be round-tripped through sqlx queries. Enum types serialize/deserialize correctly in JSON.
