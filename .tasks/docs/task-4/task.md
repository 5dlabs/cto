## Implement Design Snapshot PR Surfacing (Nova - Bun/Elysia)

### Objective
Implement the PR creation module in the PM server that creates a pull request in the 5dlabs/sigma-1 repository containing generated task scaffolds and any pipeline artifacts. Since stitch_status=failed and no design artifacts exist, the PR should contain task scaffold files generated by the pipeline. The module uses the GitHub API via the GITHUB_TOKEN secret.

### Ownership
- Agent: nova
- Stack: Bun/Elysia
- Priority: medium
- Status: pending
- Dependencies: 1

### Implementation Details
1. Create an internal module `design-snapshot` in the PM server with interface: `async function createSnapshotPR(pipelineOutput: PipelineOutput): Promise<PRResult>`.
2. Read `GITHUB_TOKEN` from environment (injected from `sigma-1-secrets`).
3. Use the GitHub API (Octokit or fetch) to: create a new branch `pipeline/<session-id>` from main in `5dlabs/sigma-1`, commit task scaffold files (one file per task, e.g., `tasks/task-<id>-<slug>.md` containing task title, description, agent, dependencies, and acceptance criteria), commit any deliberation artifacts (research memos, decision records) to a `deliberation/` directory, create a PR from the pipeline branch to main with title `[Pipeline] <session-id> - Task Scaffolds` and a body summarizing the pipeline run (task count, agent assignments, research memo availability).
4. Handle errors: if GITHUB_TOKEN is missing, log error and skip PR creation (do not fail pipeline). If repo is unreachable, retry once with exponential backoff, then log error and continue.
5. Return the PR URL for inclusion in notification payloads (Task 5).
6. Write unit tests for: branch creation, file commit structure, PR body formatting, missing token handling, API error retry.

### Subtasks
- [ ] Create design-snapshot module interface, types, and GitHub API client setup: Set up the design-snapshot module with the createSnapshotPR function signature, define PipelineOutput and PRResult types, and configure the GitHub API client with GITHUB_TOKEN from environment.
- [ ] Implement task scaffold file generation: Create the logic that transforms PipelineOutput tasks into markdown scaffold files with the correct naming convention and content structure.
- [ ] Implement GitHub branch creation and file commit logic: Implement the GitHub API calls to create a new branch pipeline/<session-id> from main and commit the generated scaffold and deliberation files to it.
- [ ] Implement PR creation with formatted body and summary: Implement the GitHub API call to create a pull request from the pipeline branch to main, with a title following the convention and a body summarizing the pipeline run.
- [ ] Implement error handling with retry logic for GitHub API failures: Add retry logic with exponential backoff for GitHub API calls and ensure the pipeline never fails due to GitHub API unavailability.
- [ ] Write integration test verifying full PR creation API call sequence: Create an integration test that runs createSnapshotPR end-to-end with a fully mocked GitHub API and verifies the complete sequence of operations.
