Implement subtask 1002: Create CloudNative-PG Cluster CR with initdb bootstrap

## Objective
Deploy the CloudNative-PG Cluster custom resource for PostgreSQL 16 with initdb bootstrap creating the sigma1 database owned by sigma1_user.

## Steps
1. Create `sigma1-postgres-cluster.yaml` with a CNPG `Cluster` CR:
   - `metadata.name: sigma1-postgres`, `metadata.namespace: sigma1`
   - `spec.instances: 1` (single replica for dev)
   - `spec.imageName`: official CNPG PostgreSQL 16 image
   - `spec.storage.size: 50Gi`
   - `spec.bootstrap.initdb.database: sigma1`
   - `spec.bootstrap.initdb.owner: sigma1_user`
   - `spec.bootstrap.initdb.secret.name: sigma1-postgres-superuser` (auto-generated by CNPG)
2. Apply the manifest.
3. Wait for the cluster to reach healthy state (this may take 2-3 minutes).
4. The post-init SQL for schemas and users is handled in the next subtask via `spec.bootstrap.initdb.postInitSQL` or `postInitApplicationSQL`.

## Validation
`kubectl get cluster sigma1-postgres -n sigma1 -o jsonpath='{.status.phase}'` returns `Cluster in healthy state`. `kubectl get pods -n sigma1 -l cnpg.io/cluster=sigma1-postgres` shows 1/1 Running pod.
