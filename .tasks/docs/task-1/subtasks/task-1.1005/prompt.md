Implement subtask 1005: MinIO Bucket Creation, Lifecycle Policies, Quotas, and Access Keys

## Objective
Create a Helm post-install hook Job that creates MinIO buckets (hermes-staging-artifacts, hermes-prod-artifacts), configures lifecycle expiry policies, sets bucket quotas, and creates per-environment access keys with scoped bucket policies. Assumes the MinIO decision point has been resolved and minio.endpoint/minio.dedicated are set in values.

## Steps
Step-by-step:
1. Create `templates/jobs/minio-bucket-setup-job.yaml`: A Kubernetes Job using `minio/mc` image.
2. Annotations: `helm.sh/hook: post-install,post-upgrade`, `helm.sh/hook-weight: "5"`, `helm.sh/hook-delete-policy: before-hook-creation`.
3. Job script:
   a. `mc alias set hermes {{ .Values.minio.endpoint }} $MINIO_ROOT_USER $MINIO_ROOT_PASSWORD`
   b. `mc mb hermes/{{ .Values.minio.bucketName }} --ignore-existing`
   c. `mc ilm add hermes/{{ .Values.minio.bucketName }} --expiry-days {{ .Values.minio.retentionDays }}`
   d. `mc admin bucket quota hermes/{{ .Values.minio.bucketName }} --hard {{ .Values.minio.bucketQuota }}`
   e. Create access key: `mc admin user add hermes {{ .Values.minio.accessKeyId }} {{ .Values.minio.secretAccessKey }}`
   f. Create and apply a bucket policy JSON granting the new user read/write access to only `{{ .Values.minio.bucketName }}`.
4. Mount MinIO root credentials from a pre-existing Secret (not the per-environment credentials being created).
5. Job must be idempotent — re-runnable without errors.
6. `restartPolicy: OnFailure`, `backoffLimit: 3`.
7. Note: MINIO_ACCESS_KEY_ID and MINIO_SECRET_ACCESS_KEY in values must be pre-generated by the operator before helm install.

## Validation
`mc ls hermes/hermes-staging-artifacts` succeeds. `mc ilm ls hermes/hermes-staging-artifacts` shows expiry rule matching retentionDays. `mc admin bucket quota hermes/hermes-staging-artifacts` shows configured quota. A test object can be PUT and GET using the per-environment access keys. PUT using staging credentials to the production bucket fails (policy isolation). Job completes with status Succeeded. Running helm upgrade a second time does not fail the Job.
