Intake Stage
Agent: Morgan (Project Management)

Morgan processes the PRD and breaks it down into atomic tasks with dependencies.

Process

  1. Create Linear Issue with prd or intake label
  2. Morgan reads PRD and researches using Firecrawl & Context7
  3. Generates task files: prompt.xml, prompt.md, acceptance_criteria.md
  4. Creates Linear project with Board view
  5. Creates child issues for each task

Acceptance Criteria

  • Tasks are atomic and independently executable
  • Dependencies between tasks are correctly identified
  • Each task has clear acceptance criteria defined
  • Linear project created with proper workflow states

Implementation Stage
Agents: Rex, Blaze, Grizz, Nova, Tap, Spark ‚Ä¢ Timeout: 30 min

Implementation agents write code based on the task prompt and create pull requests.

Acceptance Criteria

  • CodeRun resource is created successfully
  • Agent pod starts and reaches Running state
  • GitHub authentication works (token from App credentials)
  • Repository is cloned successfully
  • Task files are loaded and parsed
  • CLI (claude/cursor/code) executes without crash
  • Code changes are committed to feature branch
  • Pull request is created with proper description
  • CI checks pass on the PR

Example: Start a Play Workflow

{
  "name": "play",
  "arguments": {
    "task_id": 1,
    "repository": "5dlabs/my-project",
    "service": "my-service",
    "docs_repository": "5dlabs/my-project",
    "docs_project_directory": "my-project",
    "parallel_execution": true
  }
}
