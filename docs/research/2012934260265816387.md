# Research: claude

## Original Tweet

> forcing claude to review all of its fuck ups and create rules/skills to prevent them in the future

**Author**: @joelhooks (joelhooks)  
**Posted**: January 18, 2026 at 05:05 PM

## Analysis

This describes a highly actionable self-improvement pattern for AI agents that we could implement across our platform. The concept is: after each task/workflow failure, have the agent analyze what went wrong and generate persistent rules or skills to prevent recurrence. This is directly applicable to Rex, Blaze, Tess, and other agents. We could build: (1) A post-mortem analysis system that triggers after agent failures, (2) A structured rule/skill extraction process using Claude's reflection capabilities, (3) A persistent knowledge base where agents store learned rules, (4) A rule application layer that checks new tasks against historical failure patterns. This would significantly improve agent reliability over time through continuous learning. The implementation is straightforward - we already have error tracking, we'd add reflection prompts and a rules database. High value because it directly addresses agent reliability, which is critical for production use.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a post-mortem workflow that triggers when any agent (Rex, Blaze, Tess) fails a task, forcing it to analyze the failure
- Create a structured reflection prompt template that extracts: root cause, pattern recognition, and preventive rule generation
- Implement a persistent 'Agent Rules Database' (likely PostgreSQL or Redis) storing learned rules per agent type
- Add a pre-task validation step where agents check their accumulated rules before executing new tasks
- Build a rules management UI for human oversight - approve/reject/edit agent-generated rules
- Create metrics dashboard showing: failure rate over time, rules generated, rules applied, prevented failures
- Implement rule versioning and A/B testing to validate if rules actually prevent failures
- Add cross-agent rule sharing - if Rex learns something, can Blaze benefit from that rule?
- Build an Argo Workflow template for the reflection/rule-generation process
- Create MCP tool for 'query_learned_rules' and 'apply_rule_to_context' that agents can use
