# Research: local_llm

## Original Tweet

> Has anyone experimented with using local LLM's for codebase exploration?
> 
> I probably wouldn't trust them to write the code (not good enough) but for exploring the codebase they'd do a great job at no cost and lower latency (depending on hardware).

**Author**: @mattpocockuk (mattpocockuk)  
**Posted**: January 18, 2026 at 10:36 AM

## Analysis

This tweet presents a highly actionable implementation opportunity for our multi-agent platform. The core idea - using local LLMs for codebase exploration rather than code generation - aligns perfectly with our agent orchestration model. We could implement a specialized 'Explorer' agent or enhance existing agents (particularly Tess for testing, Rex for architecture) with local LLM capabilities for code understanding tasks. This would reduce API costs for high-frequency operations like code search, dependency analysis, and context gathering. Implementation path: (1) Integrate a local LLM runtime (llama.cpp, Ollama) as a Kubernetes service, (2) Create MCP tools for codebase querying via local LLM, (3) Build a caching layer for common queries, (4) Add agent decision logic to route exploration tasks to local LLM vs. cloud LLM based on task complexity. The cost/latency benefits are significant for our workflow-heavy platform where agents frequently need to understand code context before acting.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Deploy Ollama or llama.cpp as a Kubernetes service for local LLM inference
- Create MCP tool 'explore-codebase-local' that routes queries to local LLM
- Build a hybrid routing system: local LLM for exploration, cloud LLM for generation
- Implement caching layer for common codebase queries to further reduce latency
- Add 'CodeExplorer' agent specialized in using local LLM for code understanding tasks
- Enhance Rex (architecture agent) with local LLM for rapid dependency analysis
- Enhance Tess (testing agent) with local LLM for test coverage analysis
- Create Argo Workflow templates that use local LLM for pre-processing steps
- Build metrics dashboard comparing local vs cloud LLM performance and cost
- Implement model selection logic based on query complexity and required accuracy
