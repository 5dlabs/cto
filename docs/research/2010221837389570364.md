# Research: prompt engineering

## Original Tweet

> I ran 18 tests on a prompting technique called Code Field.
> > The prompt is 4 lines. All negations. No instructions on what to do—only what not to do.
> > Code generation results:
> Assumptions stated went from 0% to 100%. Every response listed its assumptions before writing code. Zero

## Analysis

This 'Code Field' prompting technique shows strong implementation potential for improving our AI coding agents. The concept of using only negations to guide code generation while forcing assumption declaration could significantly improve code quality and transparency. We could implement this as: 1) A prompt engineering module that transforms positive instructions into negative constraints, 2) An assumption extraction system that validates agent outputs contain explicit assumptions, 3) A prompt template library with negation-based patterns. The 100% success rate in forcing assumption declaration is particularly valuable for our multi-agent platform where transparency and explainability are crucial.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a prompt transformation service that converts positive instructions to negation-based prompts
- Implement assumption extraction and validation middleware for all coding agents
- Create a prompt template library with tested negation patterns for different coding tasks
- Add assumption tracking to agent outputs for better debugging and validation
- Develop metrics dashboard to track assumption declaration rates across agents
- Integrate negation-based prompting into Rex, Blaze, and Tess agent workflows
