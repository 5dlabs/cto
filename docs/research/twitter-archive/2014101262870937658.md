# Research: code review automation

## Original Tweet

> Devin Review has improved my review velocity >3x in the last few weeks, especially for PRs that have big refactors, which are much easier to review with copy/move detection

**Author**: @devmchheda (devmchheda)

## Analysis

This tweet highlights a specific, high-value feature for code review automation: intelligent copy/move detection in PR reviews. This is directly implementable in our platform as an enhancement to our code review agents (likely Tess or a specialized review agent). The 3x velocity improvement claim suggests significant ROI. We could build: (1) A Rust-based git diff analyzer that detects file moves/copies beyond basic git detection, (2) An MCP tool that integrates with GitHub/GitLab APIs to enhance PR review context, (3) Agent prompting strategies that specifically handle refactor PRs differently from feature PRs, (4) A review workflow that pre-processes large refactors to identify structural changes before detailed review. The technical fit is strong - we already have git integration, agent orchestration, and workflow automation. This addresses a real pain point in automated code review that our agents likely struggle with today.

**Implementation Potential**: 🔍 Worth Investigating (0.82)

### 💡 Implementation Ideas

- Build a Rust crate for advanced git diff analysis that detects file moves, copies, and structural refactors beyond standard git detection
- Create an MCP tool 'pr-analyzer' that pre-processes PRs to identify refactor patterns, copy/move operations, and structural changes
- Enhance Tess (review agent) with specialized prompting for refactor PRs that focuses on structural correctness rather than line-by-line changes
- Implement a workflow stage that classifies PRs (feature/bugfix/refactor) and routes them to appropriate review strategies
- Build a similarity detection algorithm to identify code blocks that were moved/copied across files, reducing false-positive change detection
- Create a review summary generator that highlights 'this file was moved from X to Y' vs 'this is new logic' to speed up human review
- Integrate with GitHub's 'viewed files' API to track review progress and optimize agent review ordering for large refactors
