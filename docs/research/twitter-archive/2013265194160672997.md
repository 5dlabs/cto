# Research: persistent-memory

## Original Tweet

> Anthropic is developing a new feature for Claude Cowork called Knowledge Bases (KBs) persistent, topic-specific memory containers that Claude will automatically reference and update. 
> 
> These KBs are designed to store user preferences, decisions, lessons, and facts essentially

**Author**: @WesRoth (WesRoth)  
**Posted**: January 19, 2026 at 03:00 PM

## Analysis

This describes a persistent memory/knowledge base system that we could absolutely implement for our multi-agent platform. Our agents (Rex, Blaze, Tess, etc.) currently lack persistent context about project decisions, coding standards, and learned patterns across sessions. We could build: 1) A Rust-based KB service that stores project-specific knowledge (architecture decisions, coding patterns, team preferences), 2) MCP tools for agents to query/update KBs during workflow execution, 3) Kubernetes CRDs for KB lifecycle management, 4) Auto-updating mechanisms where agents learn from code reviews and decisions. This would dramatically improve agent consistency and reduce repetitive explanations. The implementation is straightforward - we already have the infrastructure (K8s, Argo, MCP) and could use vector DBs (Qdrant/Milvus) for semantic storage. High impact for agent quality, moderate effort (2-3 weeks for MVP).

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a Rust-based Knowledge Base service with gRPC API for agent access
- Create MCP tools (kb_query, kb_update, kb_search) for agents to interact with project knowledge
- Implement Kubernetes CRD for KnowledgeBase resources with automatic persistence
- Integrate vector database (Qdrant/Milvus) for semantic search of decisions and patterns
- Add workflow hooks in Argo to auto-update KBs after code reviews or deployments
- Create KB templates for different project types (microservices, libraries, APIs)
- Build KB versioning system to track evolution of project decisions over time
- Implement KB sharing/inheritance between related projects or teams
