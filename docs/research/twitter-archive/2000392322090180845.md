# Research: multi-model-systems

## Original Tweet

> we have a /commit slash command and i just recently thought to set it to a fast/cheap model
> 
> that way i can be in the middle of using any model and /commit will still use the right one

**Author**: @thdxr (thdxr)  
**Posted**: December 15, 2025 at 02:27 AM

## Analysis

This tweet describes a highly actionable UX pattern for multi-model AI systems: context-specific model routing based on command type. The core insight is that different operations have different cost/speed/quality tradeoffs - commits need speed and low cost, while complex refactoring needs powerful models. We could implement this in our platform as: (1) Command-specific model routing in our agent orchestration layer, (2) A configuration system mapping operation types (commit, review, refactor, test) to appropriate models, (3) Cost optimization by using cheaper models for routine tasks while preserving quality for complex ones. This directly addresses a real pain point in multi-agent systems where you want flexibility to use different models but don't want to manually switch contexts. Implementation would involve extending our agent configuration to support operation-type-based model selection, potentially saving significant API costs while maintaining quality where it matters.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Add operation-type-based model routing to agent configuration (commit operations use fast/cheap models like GPT-3.5-turbo or Claude Haiku)
- Create a model selection strategy system where each agent operation type (commit, review, refactor, test, debug) maps to appropriate model tier
- Implement slash-command or operation-type detection in our agent orchestration layer to automatically route to cost-optimized models
- Build a cost tracking system that shows savings from using tiered model selection vs always using premium models
- Add configuration DSL for defining model routing rules: 'commit: fast', 'architecture: premium', 'test: balanced'
- Create agent profiles with default model preferences per operation type that users can override
