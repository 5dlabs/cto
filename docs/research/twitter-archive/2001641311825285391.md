# Research: event-streaming

## Original Tweet

> Built this last night....... Claude Code Matrix Monitor hooks into Claude Code's native event system to visualize every tool call in real-time. 
> 
> All of Claude's file reads, searches, edits, and bash commands stream via WebSocket to a cyberpunk dashboard that renders radar

**Author**: @BenjaminDEKR (BenjaminDEKR)  
**Posted**: December 18, 2025 at 01:10 PM

## Analysis

This is highly actionable for our platform. The concept of hooking into AI agent event systems to visualize tool calls in real-time directly applies to monitoring our multi-agent orchestration. We could implement: 1) A real-time monitoring dashboard for Rex, Blaze, Tess, and other agents showing their MCP tool calls, file operations, and workflow steps, 2) WebSocket-based event streaming from our agents to a centralized monitoring UI, 3) Integration with our existing Argo Workflows to visualize agent execution paths, 4) A debugging/observability layer that helps developers understand what agents are doing. This would significantly improve platform transparency, debugging capabilities, and developer experience. The technical approach (event system hooks + WebSocket streaming + visualization) is directly implementable in our Rust backend with React frontend. The 'cyberpunk dashboard' aesthetic is less important than the core monitoring architecture, which solves a real need for agent observability.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a centralized event bus in Rust that captures all agent tool calls (MCP operations, file reads/writes, bash commands, API calls)
- Create WebSocket streaming service to push agent events to monitoring dashboard in real-time
- Develop React-based monitoring UI showing live agent activity across all running workflows
- Implement event hooks in each agent (Rex, Blaze, Tess, Nova) to emit structured events for all operations
- Add filtering and search capabilities to monitor specific agents, workflows, or tool types
- Integrate with Argo Workflows to correlate agent events with workflow execution stages
- Build replay functionality to review past agent execution patterns for debugging
- Create alerting system for anomalous agent behavior or failed tool calls
