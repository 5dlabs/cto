# Research: slash-commands

## Original Tweet

> We heard slash commands were hot right now, so we added one for every touchpoint in Warp 
> 
> /usage, /conversations, /add-rule, /view-mcp, /add-prompt, and many more. What else should we add?

**Author**: @warpdotdev (warpdotdev)  
**Posted**: December 13, 2025 at 02:15 PM

## Analysis

This tweet presents a highly actionable pattern for our multi-agent platform: implementing a slash command interface for agent orchestration and platform management. Warp's approach of creating dedicated commands for different touchpoints (/usage, /conversations, /add-rule, /view-mcp, /add-prompt) directly maps to our needs. We could implement: 1) A slash command parser in our CLI/UI layer, 2) Commands for agent management (/deploy-agent, /agent-status, /agent-logs), 3) Workflow commands (/run-workflow, /workflow-status, /cancel-workflow), 4) MCP tool commands (/list-tools, /add-mcp-server, /test-tool), 5) Context management (/add-context, /view-context), 6) Debug commands (/trace-execution, /view-metrics). The /view-mcp command is particularly relevant since we use MCP extensively. This would significantly improve developer experience by providing a unified, discoverable interface for platform operations. Implementation would involve: building a command registry in Rust, creating a parser, integrating with our existing API layer, and adding autocomplete/help functionality. The pattern is proven (Warp, Slack, Discord all use it successfully) and would reduce cognitive load for developers interacting with our complex multi-agent system.

**Implementation Potential**: 🔍 Worth Investigating (0.82)
