# Research: TDD

## Original Tweet

> hates hate it but TDD is fuckin awesome when you are swarming coding agents

**Author**: @joelhooks (joelhooks)  
**Posted**: December 13, 2025 at 05:01 PM

## Analysis

This tweet highlights a critical implementation gap in our multi-agent platform: systematic TDD integration for agent-generated code. The insight that TDD becomes 'awesome' when coordinating multiple coding agents suggests we could implement: (1) Test-first workflow orchestration where agents generate tests before implementation, (2) Agent-specific test generation capabilities (e.g., Tess could specialize in test creation), (3) Test-driven agent coordination patterns where one agent writes tests and another implements, (4) Automated test coverage enforcement in our Argo Workflows, (5) MCP tools for test generation and validation. This is highly actionable because we already have the agent infrastructure - we just need to formalize TDD patterns into our orchestration layer. The 'swarming' aspect suggests parallel test generation and validation could significantly improve code quality when multiple agents collaborate.

**Implementation Potential**: 🔍 Worth Investigating (0.82)

### 💡 Implementation Ideas

- Create a dedicated test-generation agent (or enhance Tess) that writes tests before implementation agents start coding
- Build an Argo Workflow template for TDD cycles: test generation → implementation → validation → refactor
- Implement MCP tool 'generate_tests_for_spec' that agents can call to create comprehensive test suites
- Add test-first orchestration mode where Rex coordinates: Tess writes tests, Blaze implements, Nova validates
- Create Kubernetes operator that enforces test coverage thresholds before allowing agent-generated code to merge
- Build agent collaboration pattern: one agent proposes tests, another reviews/enhances them, third implements
- Integrate test mutation tools to validate that agent-generated tests actually catch bugs
- Add workflow step that requires passing tests before subsequent agents can proceed with their tasks
