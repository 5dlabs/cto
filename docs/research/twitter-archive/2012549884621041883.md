## Analysis

The Expo Skills repository contains AI-powered development skills/tools that could serve as excellent reference implementations for our MCP tools. These are structured, actionable patterns for common development tasks (upgrades, debugging, testing) that our agents could adopt. The 'upgrade' skill is particularly relevant - we could build similar MCP tools that help our agents automatically upgrade dependencies, migrate codebases, or apply framework updates. The repository likely contains prompt engineering patterns, tool definitions, and workflow structures that would directly inform how we build our agent capabilities. This is highly actionable because: 1) It's open source with implementation details, 2) Skills-based architecture aligns with our MCP tool approach, 3) We could either integrate these directly or build similar tools for our tech stack (Rust, K8s, etc.). Not 0.9+ because it's React Native/Expo specific, so we'd need to adapt rather than directly use, but the patterns and architecture are highly transferable.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build MCP tools modeled after Expo Skills for Rust crate upgrades, Kubernetes manifest updates, and dependency migrations
- Create a 'skills registry' pattern where agents can discover and invoke specialized tools for common development tasks
- Implement automated upgrade workflows that analyze breaking changes and apply fixes (similar to Expo's upgrade skill)
- Develop prompt templates and tool definitions based on their skill structure for our agents (Rex, Blaze, Tess)
- Build a debugging skill for our platform that helps agents diagnose and fix common Rust compilation errors or K8s deployment issues
- Create testing skills that generate and run tests for code changes made by our agents

# Expo Skills

Official AI agent skills from the Expo team for building, deploying, and debugging robust Expo apps.

## Installation

[Permalink: Installation](https://github.com/expo/skills#installation)

We primarily use [Claude Code](https://claude.com/claude-code) at Expo, skills are fine-tuned for Opus models. But you can use these skills with any AI agent.

## Claude Code

[Permalink: Claude Code](https://github.com/expo/skills#claude-code)

Add the marketplace:

```
/plugin marketplace add expo/skills
```

Install a plugin:

```
/plugin install expo-app-design
/plugin install upgrading-expo
/plugin install expo-deployment
```

## Cursor

[Permalink: Cursor](https://github.com/expo/skills#cursor)

**Install from GitHub**

1. Open Cursor Settings (Cmd+Shift+J / Ctrl+Shift+J)
2. Navigate to `Rules & Command` → `Project Rules` → Add Rule → Remote Rule (GitHub)
3. Enter: `https://github.com/expo/skills.git`

**How it works:** Skills are automatically discovered and used by the agent based on context. When you ask questions about Expo development, the agent will automatically use the relevant skills (e.g., `building-ui`, `data-fetching`, `deployment`) based on the skill descriptions.

**Note:** Skills won't appear in the `/` slash command menu. The `/` menu in Cursor is for custom commands (stored in `.cursor/commands/`), not for skills. Skills work via auto-discovery - the agent uses them automatically when your questions match their descriptions.

**Verify installation:** After adding the Remote Rule, try asking the agent Expo-related questions like:

- "How do I build a UI with Expo Router?"
- "How do I make API calls in my Expo app?"
- "How do I deploy my Expo app to the App Store?"

If the skills are working, the agent will use the relevant skill content to answer your questions.
