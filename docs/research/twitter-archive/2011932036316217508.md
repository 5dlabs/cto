## Analysis

This tweet points to a repository containing structured 'skills' and 'rules' for AI coding agents (Claude/Cursor) working with Convex. The HIGH VALUE here is the PATTERN of packaging reusable agent skills/rules - we could adopt this approach for our Rex/Blaze/Tess agents. Specifically: (1) Create similar skill packages for our tech stack (Rust, K8s, Argo), (2) Structure agent context/prompts as reusable modules, (3) Build an MCP tool that loads context-specific skills dynamically based on project type. The Convex-specific content is less relevant since we don't use it, but the META-PATTERN of codifying agent expertise into installable packages is directly applicable. The GitHub repo likely contains the actual skill definitions and rule structures we could adapt - worth scraping to see HOW they've structured this knowledge.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Create 'RexSkills', 'BlazeSkills', 'TessSkills' packages containing agent-specific expertise for Rust, K8s, Argo workflows
- Build an MCP tool that dynamically loads skill packages based on project context (e.g., 'rust-backend-skills', 'k8s-operator-skills')
- Develop a skill package format/schema for our platform that agents can consume to enhance their capabilities
- Create npm-installable (or cargo-installable) skill packages that teams can add to projects to improve agent performance
- Implement a skill registry service that agents query to get context-appropriate expertise
- Extract the pattern of how they structure rules/skills and apply it to our agent system prompts and MCP tool configurations
