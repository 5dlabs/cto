# Research: agent skills management

## Original Tweet

> A step towards unifying Agent Skills: 
> 
> The latest version of `npx skills add` now copies Skill files to .agents/ (project or global)
> 
> and symlinks to all the agent specific folders (.config, .claude, .codex, etc)
> 
> making it easy to update skills in 1 place

**Author**: @andrewqu (andrewqu)  
**Posted**: January 21, 2026 at 03:43 AM

## Analysis

This tweet describes a practical pattern for managing shared agent skills/tools across multiple AI agents - directly applicable to our multi-agent platform (Rex, Blaze, Tess, etc.). The approach of centralizing skill definitions in a single location (.agents/) and symlinking to agent-specific directories solves a real problem: keeping agent capabilities synchronized without duplication. We could implement a similar system for our MCP tools and agent configurations, potentially using Rust for the CLI tooling and leveraging our Kubernetes infrastructure for distribution. The pattern is especially relevant since we already use MCP tools - this would improve how we manage and update those tools across our agent fleet. Implementation would involve: (1) creating a centralized skill/tool registry, (2) building a CLI tool (possibly 'cto-skills' or integrating into existing tooling), (3) implementing symlink or mount strategies in our K8s pods, (4) version control for skill definitions. The main limitation is this appears to be a Node.js tool, but the pattern itself is language-agnostic and highly valuable.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a 'cto skills' CLI in Rust for managing shared agent capabilities across Rex, Blaze, Tess, etc.
- Create a centralized skill registry in our platform with version control and rollback capabilities
- Implement ConfigMap or Volume mounting strategy in Kubernetes to distribute skills to agent pods
- Design a skill definition format (YAML/TOML) that works with our MCP tool architecture
- Add skill synchronization to our Argo Workflows for automated updates across the agent fleet
- Build a skill marketplace or repository where teams can share custom MCP tools and agent capabilities
- Implement symlink strategy in agent containers or use K8s projected volumes for skill sharing
- Create skill versioning and dependency management system to prevent conflicts between agents
