# Research: token optimization

## Original Tweet

> We always aim for minimal context usage @Context7AI 
> 
> - our tool descriptions are just 900 tokens
> - only 0.5% of the context window on Opus 4.5
> 
> We’ve tested larger toolsets many times but performance drops 
> 
> At least until now 

**Author**: @abdushbag (abdushbag)  
**Posted**: January 15, 2026 at 12:06 AM

## Analysis

This tweet reveals a critical optimization pattern for AI agent systems: token-efficient tool descriptions. Context7AI has empirically found that keeping tool descriptions to ~900 tokens (0.5% of context window) maintains performance while larger toolsets degrade it. This is HIGHLY actionable for our multi-agent platform where Rex, Blaze, Tess, etc. use MCP tools. We could: 1) Audit our current MCP tool descriptions for token usage, 2) Implement a token budget system for tool definitions, 3) Create compression/optimization utilities for tool schemas, 4) Build monitoring to track context usage vs agent performance correlation. The specific numbers (900 tokens, 0.5% threshold) give us concrete targets. This directly impacts our agent orchestration efficiency and could reduce costs while improving response quality.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a token budget analyzer for MCP tool descriptions - scan all our tools and flag those exceeding 900 tokens
- Create a tool description optimizer that compresses verbose schemas while preserving functionality
- Implement context usage monitoring in Argo Workflows to correlate tool count/size with agent performance metrics
- Develop a 'minimal tool description' linter that enforces token limits during tool registration
- Add telemetry to track context window utilization across different agent types (Rex vs Blaze vs Tess)
- Build a tool description template system that enforces concise, token-efficient patterns
- Create an A/B testing framework to validate the 0.5% threshold for our specific agent workloads
