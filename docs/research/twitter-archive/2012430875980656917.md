# Research: authentication

## Original Tweet

> Coding agents can now roll their own auth 

**Author**: @andrewqu (andrewqu)  
**Posted**: January 17, 2026 at 07:44 AM

## Analysis

This tweet suggests coding agents can now handle authentication flows autonomously - a critical capability for our multi-agent platform. Currently, our agents (Rex, Blaze, Tess) likely require pre-configured credentials or manual auth setup. If agents can 'roll their own auth', this could mean: (1) Agents dynamically generating API keys/tokens, (2) Implementing OAuth flows programmatically, (3) Managing service account creation, or (4) Handling auth rotation/renewal. This is highly actionable because authentication is a constant friction point in automated development - agents need to interact with GitHub, cloud providers, databases, etc. Implementation would involve building MCP tools for auth management, creating Rust libraries for common auth patterns, and potentially developing a secure credential vault that agents can programmatically access. The vague nature requires investigation to understand the specific mechanism, but the problem space is directly relevant to our orchestration challenges.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build MCP tool for dynamic OAuth flow handling that agents can invoke
- Create Rust crate for agent-driven service account provisioning across cloud providers
- Implement secure credential vault with programmatic agent access via MCP
- Develop Argo Workflow templates for auth setup/teardown in agent tasks
- Add agent capability to generate and rotate API keys for services they create
- Build authentication middleware that agents can configure declaratively
- Create MCP tool for GitHub App installation and token management by agents
