# Research: feedback-loops

## Original Tweet

> I keep improving my agentic feedback loop.

> A feedback loop is critical because it helps the coding agent see if it made any mistakes.
> 
> My feedback loop is an npm script called 'check'.
> 
> It *concurrently* runs:
> - tests
> - lint
> - spell check
> - prettier
> - build
> 
> Takes ~5 seconds.

**Author**: @housecor (housecor)  
**Posted**: January 21, 2026 at 10:17 PM

## Analysis

This describes a highly actionable feedback loop pattern that we should implement for our coding agents. The concept of a concurrent validation pipeline (tests, lint, spell check, formatting, build) running in ~5 seconds is directly applicable to our multi-agent platform. We could build this as: (1) A Rust-based validation orchestrator that runs checks concurrently, (2) An Argo Workflow template for post-generation validation, (3) An MCP tool that agents can invoke to validate their code changes, (4) A feedback mechanism that allows agents to self-correct based on validation results. The 5-second execution time is critical for agent iteration speed. This would significantly improve code quality and reduce human review cycles. While the tweet mentions npm/JS tooling, the pattern is language-agnostic and we'd implement it for our supported languages (Rust, Python, TypeScript, etc.).

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a Rust-based concurrent validation runner that executes language-specific checks (cargo test, cargo clippy, rustfmt, cargo build) in parallel with configurable timeouts
- Create an Argo Workflow template 'validation-pipeline' that agents trigger after code generation to get immediate feedback
- Develop an MCP tool 'validate_code' that agents can call to run all checks and receive structured feedback (pass/fail, error messages, suggestions)
- Implement a feedback loop in agent prompts where validation results are fed back to the agent for self-correction before human review
- Build a validation result parser that converts tool outputs (test failures, lint warnings, build errors) into agent-readable feedback format
- Create a validation cache layer to skip unchanged checks and maintain the ~5 second target
- Add validation metrics to agent performance tracking (validation pass rate, iterations to clean code, time to validation)
- Extend the pattern to support multi-language projects with language-specific validation tools running concurrently
