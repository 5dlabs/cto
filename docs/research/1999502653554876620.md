# Research: agent memory

## Original Tweet

> Memory for your @aisdk agents!

**Author**: @nicoalbanese10 (nicoalbanese10)  
**Posted**: December 12, 2025 at 03:32 PM

## Analysis

Memory capabilities for AI agents is a high-value feature we could implement. The @aisdk (Vercel AI SDK) has memory primitives that could inform how we add persistent context and conversation history to our coding agents (Rex, Blaze, Tess). This would enable agents to maintain context across workflow steps, remember previous decisions, and provide more coherent multi-turn interactions. While we'd need to adapt patterns from TypeScript/Node to our Rust stack, the core concepts (memory stores, context windows, retrieval strategies) are language-agnostic. This directly addresses a likely gap in our current agent architecture - stateless agents that don't retain context between invocations.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a memory layer for our agents using Redis or PostgreSQL to persist conversation context across workflow steps
- Implement context window management to selectively include relevant past interactions when invoking agents
- Create a Rust-based memory store abstraction that agents can use to save/retrieve decisions and code context
- Add memory-aware MCP tools that can query past agent interactions and decisions
- Design a memory retrieval strategy (semantic search, recency-based, or hybrid) for agent context
- Integrate memory checkpointing into Argo Workflows so agents can resume with full context after failures
