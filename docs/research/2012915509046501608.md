# Research: agent-autonomy

## Original Tweet

> Never add file tree structure to your agents md files. It’s a bad idea.
> 
> This is generally a core service of every coding agent that they can discover things on their own. 
> 
> File structure is the first thing that will go stale. You don’t want that. 
> 
> So many different strategies

**Author**: @MrAhmadAwais (MrAhmadAwais)  
**Posted**: January 18, 2026 at 03:50 PM

## Analysis

This tweet addresses a critical architectural decision for our agent system: how agents discover and understand codebases. The core insight is that static file trees in documentation become stale and agents should dynamically discover project structure. This is highly actionable for our platform. We could implement: (1) Dynamic file discovery services that agents query in real-time rather than relying on static docs, (2) Caching strategies for file structure that auto-invalidate on filesystem changes, (3) Agent capabilities to traverse and index codebases on-demand, (4) MCP tools for filesystem exploration that provide fresh data. The 'many different strategies' hint suggests there are multiple implementation patterns we should research. This directly impacts how Rex, Blaze, and other coding agents interact with repositories. The implementation effort is moderate (likely 1-2 weeks for basic dynamic discovery, more for sophisticated caching), but the impact is high as it prevents documentation drift and improves agent autonomy.

**Implementation Potential**: 🔍 Worth Investigating (0.82)

### 💡 Implementation Ideas

- Build an MCP tool for real-time filesystem discovery that agents can query instead of reading static file trees
- Implement a caching layer for project structure that auto-invalidates on git commits or filesystem changes
- Create agent capabilities for autonomous codebase traversal and indexing on-demand
- Develop a filesystem watcher service that keeps agent context fresh without manual documentation updates
- Add agent prompts/instructions that discourage static file tree documentation in favor of dynamic discovery
- Build a project structure API that provides versioned, timestamped file trees with staleness indicators
- Integrate filesystem exploration into agent planning phase so they always work with current state
