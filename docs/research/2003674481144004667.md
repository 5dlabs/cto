# Research: agent-patterns

## Original Tweet

> After writing several agents with Claude Code SDK, OpenAI SDKs and DeepAgent SDK I think a majority of agents can be built by just prompts and tools! 
> 
> You don’t need to implement patterns like plan, reflect and tool usage manually anymore.

**Author**: @diptanu (diptanu)  
**Posted**: December 24, 2025 at 03:49 AM

## Analysis

This tweet highlights a significant architectural simplification opportunity for our agent platform. The author's experience across multiple SDKs (Claude Code SDK, OpenAI SDKs, DeepAgent SDK) suggests that modern LLM providers have internalized complex agentic patterns (planning, reflection, tool usage) into their APIs. This means we could potentially SIMPLIFY our agent orchestration layer by relying more on provider-native capabilities rather than implementing these patterns ourselves in Rust. IMPLEMENTATION POTENTIAL: (1) Evaluate if we're over-engineering agent logic that could be handled by prompt engineering + tool definitions, (2) Create a comparative analysis of which patterns we should keep in our orchestration vs delegate to LLM providers, (3) Build a simplified agent template that uses provider-native planning/reflection, (4) Potentially reduce our Rust codebase complexity while maintaining or improving agent capabilities. This could significantly reduce maintenance burden and improve agent performance by leveraging provider optimizations. Worth investigating the mentioned SDKs to understand what patterns they've internalized.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Audit our current agent implementation to identify which planning/reflection/tool-usage patterns we've manually coded that might be redundant with provider capabilities
- Create a proof-of-concept agent using only prompts + tool definitions (no manual pattern implementation) to compare performance and complexity
- Build an abstraction layer that can switch between manual pattern implementation and provider-native patterns based on configuration
- Develop a migration guide for converting our existing agents (Rex, Blaze, Tess, Nova) to use simpler prompt+tool approaches where applicable
- Integrate Claude Code SDK or DeepAgent SDK as alternative agent runtimes to compare against our current Rust implementation
- Create a decision matrix tool that helps determine when to use provider-native patterns vs custom implementation for new agents
