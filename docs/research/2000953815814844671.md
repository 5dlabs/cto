# Research: MCP

## Original Tweet

> install md:  SaaS for installing complex software as MCP

**Author**: @bibryam (bibryam)  
**Posted**: December 16, 2025 at 03:39 PM

## Analysis

This tweet references 'install md' - a SaaS for installing complex software as MCP (Model Context Protocol). This is highly relevant because: 1) We already use MCP tools extensively in our platform, 2) Installing and managing complex software dependencies is a common pain point in multi-agent development workflows, 3) We could either integrate with this service OR build our own MCP-based installation orchestrator for our agents. The concept of wrapping software installation as MCP tools would allow our agents (especially Rex for infrastructure, Tess for testing environments) to programmatically install and configure dependencies. Implementation could include: building MCP tools that wrap package managers (apt, brew, cargo), creating an installation orchestration layer that agents can call via MCP, or developing a catalog of installation recipes accessible through MCP. This directly addresses a gap in our current tooling where agents often need manual intervention for complex software setup.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build MCP tools that wrap common package managers (cargo, apt, brew, npm) for agent-driven installation
- Create an 'install orchestrator' agent that specializes in dependency management via MCP
- Develop a catalog of installation recipes (databases, tools, services) accessible through MCP protocol
- Integrate with or build similar SaaS for managing complex software installations in our Kubernetes environments
- Add MCP-based installation capabilities to Rex (infrastructure agent) for automated environment setup
- Create MCP tools for Docker/container image building and management
- Build validation MCP tools that verify installations succeeded (useful for Tess in testing)
