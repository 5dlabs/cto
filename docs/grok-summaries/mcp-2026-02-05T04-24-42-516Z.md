## Summary

**Model Context Protocol (MCP)** is an open standard developed by Anthropic (now under the Linux Foundation) that standardizes how AI models like Claude connect to external data sources, tools, databases, APIs, file systems, and SaaS applications—often described as "USB-C for AI" or the "new API for agents."[[1]](https://x.com/i/status/2018225374786732247)[[2]](https://x.com/i/status/2018625550492094865)[[3]](https://x.com/i/status/2018354138258768005)

### Key Features:
- **Runtime Discovery**: AI can dynamically query available tools via `tools/list` without hardcoding or regenerating SDKs.[[1]](https://x.com/i/status/2018225374786732247)
- **Deterministic Execution**: LLMs select tools, but actual execution uses wrapped, reliable code—avoiding hallucinations in API calls.[[1]](https://x.com/i/status/2018225374786732247)
- **Bidirectional Communication**: Servers can request completions from the LLM, seek user input, or send progress updates.[[1]](https://x.com/i/status/2018225374786732247)
- **Unified JSON Schema**: Simplifies inputs/outputs compared to scattered REST params.[[1]](https://x.com/i/status/2018225374786732247)
- **Local-First Design**: Runs over stdio for local tools, inheriting host permissions for filesystem access, no ports/CORS issues.[[1]](https://x.com/i/status/2018225374786732247)

### Adoption & Ecosystem:
- Supported by OpenAI, Microsoft, Google.[[1]](https://x.com/i/status/2018225374786732247)[[2]](https://x.com/i/status/2018625550492094865)
- Native in tools like Claude Desktop, Cursor, Windsurf; Zapier exposes 8,000+ apps via MCP.[[1]](https://x.com/i/status/2018225374786732247)
- Open-source implementations: mcp-go (8k+ stars), servers for GitHub, Slack, Postgres, Figma, Blender, etc.[[4]](https://x.com/i/status/2018464205016633451)[[5]](https://x.com/i/status/2017889406791594103)
- Released ~Nov 2024, now 10,000+ MCP servers in production.[[6]](https://x.com/i/status/2018399021447135652)[[2]](https://x.com/i/status/2018625550492094865)

### Caveats:
Security relies on server-side implementation (auth, validation)—no protocol-level enforcement.[[1]](https://x.com/i/status/2018225374786732247)
