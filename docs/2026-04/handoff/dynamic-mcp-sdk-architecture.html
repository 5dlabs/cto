🔧 Dynamic MCP Tool SDK

Anthropic "Code Execution with MCP" pattern · PR #4610

🎯 The Objective

  • Agents discover & call 150+ MCP tools without loading schemas into context
  • Agents write TypeScript that imports only needed tools → runs in Deno sandbox
  • Intermediate data (10K rows → 5) stays in execution env, never enters model context
  • Works across all 6 runtimes: Claude Code, Codex, OpenCode, Factory, Gemini, Cursor
  • Escalation engine controls which tools each agent session can access (allow/deny globs)
  • 98.7% token savings on large tool catalogs (Anthropic's measured result)

❌ Before (Direct Tool Loading)

150 tool schemas loaded upfront ~200K tokens

Every tool call result flows through model context. A 2-hour meeting transcript through 2 tool calls = ~50K wasted tokens.

✅ After (Code Execution Mode)

File tree listing + agent-written TS ~2.5K tokens

Agent sees directory tree, imports what it needs, data flows through Deno sandbox. Model only sees final result.
