# Research: agent design

## Original Tweet

> the trick to creating effective SKILLS is to make your ai *think* like an expert, not just follow steps

**Author**: @boringmarketer (boringmarketer)  
**Posted**: December 24, 2025 at 03:14 PM

## Analysis

This tweet highlights a critical architectural principle for AI agent design that we could implement across our agent system. The distinction between 'thinking like an expert' vs 'following steps' suggests we should enhance our agent prompting strategy with expert reasoning patterns, mental models, and decision-making frameworks rather than just procedural instructions. This is highly actionable for improving Rex, Blaze, Tess, and other agents. We could: 1) Refactor agent system prompts to include expert mental models (e.g., Rex thinks like a senior architect considering tradeoffs, Blaze thinks like a performance engineer analyzing bottlenecks), 2) Implement a 'reasoning layer' that makes agents explain their thought process before acting, 3) Create agent-specific 'expertise profiles' that define how each agent should approach problems, 4) Build a prompt engineering framework that structures expert thinking patterns. This directly impacts agent quality and would differentiate our platform by producing more thoughtful, context-aware code generation.

**Implementation Potential**: 🔍 Worth Investigating (0.82)

### 💡 Implementation Ideas

- Refactor agent system prompts to include expert mental models and reasoning frameworks specific to each agent's domain (architecture, testing, deployment, etc.)
- Build a 'reasoning layer' middleware that requires agents to articulate their thought process and decision rationale before executing actions
- Create structured 'expertise profiles' for each agent defining how they should analyze problems, what they should consider, and how they should make decisions
- Implement a prompt template system that embeds expert thinking patterns (e.g., 'consider tradeoffs', 'identify edge cases', 'evaluate alternatives')
- Add agent introspection capabilities where agents can explain WHY they chose a particular approach, not just WHAT they did
- Develop a library of domain-specific mental models that agents can reference (e.g., performance optimization heuristics, security threat modeling, architectural patterns)
- Create agent evaluation metrics that measure quality of reasoning, not just correctness of output
