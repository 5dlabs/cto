# Research: Docker

## Original Tweet

> Out now in Sandbox SDK v0.6.5 
> You can bring your own Dockerfile!
> 
> We still publish 3 base images to use as-is or extend. But if you have an existing setup, adding the Sandbox binary is now only 3 lines. The binary will also take care of starting your custom scripts as well.

**Author**: @ghostwriternr (ghostwriternr)  
**Posted**: December 12, 2025 at 11:57 AM

## Analysis

This is highly relevant to our multi-agent platform. The Sandbox SDK v0.6.5's 'bring your own Dockerfile' feature directly addresses a key challenge in agent orchestration: providing flexible, customizable execution environments. We could implement: (1) A similar pattern for our agents to run in custom containers while maintaining platform integration, (2) A lightweight binary injection pattern for our MCP tools to work in any container, (3) Custom environment support per agent type (Rex might need different tools than Tess). The 3-line integration approach suggests a clean architecture we could adopt. This would significantly improve our platform's flexibility - currently our agents likely run in standardized containers, but customers may want custom dependencies, tools, or configurations. The 'binary takes care of starting custom scripts' pattern is particularly valuable for our workflow orchestration.

**Implementation Potential**: 🔍 Worth Investigating (0.85)

### 💡 Implementation Ideas

- Build a 'platform-agent-runtime' binary that can be injected into any Dockerfile with 3 lines, similar to Sandbox SDK approach
- Create base images for each agent type (Rex, Blaze, Tess) that customers can extend with their own dependencies
- Implement a container customization API that allows users to specify custom Dockerfiles per agent while maintaining platform integration
- Develop a binary that handles both platform communication (MCP, orchestration) and custom script execution in user-defined containers
- Add support for custom init scripts that run before agent execution, managed by our platform binary
- Create a validation system to ensure custom Dockerfiles meet minimum platform requirements (network access, file permissions, etc.)
- Build a container registry integration for storing and versioning custom agent images
