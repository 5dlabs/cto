## Analysis

This tweet describes a highly actionable technique for improving AI-generated code quality in our platform. The core idea is to use Wikipedia's documented 'signs of AI writing' as negative examples to train our coding agents (Rex, Blaze, Tess, etc.) to produce more natural, human-like code and documentation. We could implement this as: 1) A prompt engineering enhancement layer that preprocesses agent instructions with 'avoid these patterns' guidelines, 2) A post-processing validation step that checks generated code/docs against AI writing telltales, 3) A new MCP tool that fetches and parses the Wikipedia article to dynamically update our anti-patterns list, 4) Integration into our code review workflows where Tess could flag AI-like patterns. The implementation effort is moderate (likely 1-2 weeks) but the impact is significant - improving code quality and making our agents' output less detectable as AI-generated could be a competitive advantage. The technique is stack-agnostic and applies to all our agents regardless of whether they're generating Rust, React, or documentation.

**Implementation Potential**: 🔍 Worth Investigating (0.82)

### 💡 Implementation Ideas

- Create an MCP tool that scrapes Wikipedia's 'AI writing signs' article and converts it into structured anti-patterns
- Build a prompt preprocessing layer that injects 'avoid these AI telltales' instructions into all agent prompts
- Implement a post-generation validator that scores code/docs against known AI writing patterns
- Add a Tess code review check that flags potential AI-like patterns in generated code
- Create a reusable 'humanize output' skill/prompt template that all agents can inherit
- Build a feedback loop where flagged AI patterns are added to our anti-pattern database
- Integrate this as a quality gate in our Argo Workflows before code commits
