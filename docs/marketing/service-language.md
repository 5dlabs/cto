# CTO Service Language

This document captures the customer-facing service names now used across the CTO marketing site.

## Goal

Keep public-facing copy focused on outcomes, not operator names, internal implementation details, or infrastructure brand soup.

## Naming Map

- `5D Command`: the connected control layer across code, tasks, infrastructure, and alerts.
- `5D Flow`: planning, task sync, and live delivery coordination.
- `5D Deploy`: release flow, deployment automation, and rollback handling.
- `5D Guard`: health checks, incident detection, and automated remediation.
- `5D Vault`: secrets, credentials, and bring-your-own-key workflows.
- `5D Observe`: metrics, logs, traces, and alerting.
- `5D Store`: object and file storage.
- `5D Volume`: replicated high-speed persistent volumes.
- `5D Inference`: hosted or self-hosted model runtime behind a consistent interface.
- `5D Postgres`: customer-facing shorthand for the managed relational data layer.

## Copy Rules

- Prefer outcome language first.
- Mention the branded service name before any technical implementation detail.
- Only expose underlying vendor or open source names when the audience explicitly needs them.
- Keep model lists and infrastructure internals off the main homepage unless they directly support a buying decision.
