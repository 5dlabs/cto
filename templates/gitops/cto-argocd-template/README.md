# CTO ArgoCD GitOps Repository

This repository contains the ArgoCD Application manifests for your CTO Managed Dedicated cluster.

## Overview

This repo is automatically created and managed by the CTO platform. It contains:

- **ArgoCD Application manifests** - Declarative configuration for what should run on your cluster
- **Tenant-specific values** - Configuration specific to your organization
- **Runtime component definitions** - Agent controller, MCP tools, and other platform services

## Structure

```
.
├── README.md                    # This file
├── values.yaml                  # Tenant-specific configuration
├── applications/
│   ├── app-of-apps.yaml        # Root ArgoCD Application (manages all others)
│   ├── agent-controller.yaml   # CTO Agent Controller
│   ├── mcp-tools.yaml          # MCP Tool sidecars
│   ├── cilium.yaml             # Cilium CNI with ClusterMesh
│   └── local-path.yaml         # Local path provisioner for storage
└── base/
    ├── namespaces.yaml         # Namespace definitions
    └── rbac.yaml               # RBAC configurations
```

## How It Works

1. **ArgoCD on your cluster** watches this repository
2. When changes are pushed, ArgoCD automatically syncs them to your cluster
3. The platform team can push updates (security patches, new features) via this repo
4. You can customize `values.yaml` to adjust configurations

## Customization

### Adjusting Runtime Configuration

Edit `values.yaml` to customize:

```yaml
tenant:
  id: your-tenant-id
  provider: latitude
  region: DAL

cluster:
  name: your-cluster-name

applications:
  - name: agent-controller
    enabled: true
  - name: mcp-tools
    enabled: true
```

### Disabling Components

To disable a component, set `enabled: false`:

```yaml
applications:
  - name: some-component
    enabled: false
```
