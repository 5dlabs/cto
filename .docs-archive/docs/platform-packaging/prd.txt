# CTO Platform-in-a-Box: Product Requirements Document

## Executive Summary

The CTO Platform-in-a-Box is a productized, distributable version of the 5D Labs CTO
platform delivered as a bootable ISO image. It enables organizations to deploy a complete
cloud-native development platform on bare-metal hardware with minimal technical expertise.

**Vision**: "Engineering team infrastructure on a USB key"

**Target Users**: Development teams and small-to-medium organizations who want on-premises
cloud infrastructure without dedicated DevOps staff.

---

## Problem Statement

Organizations face significant challenges deploying modern cloud-native infrastructure:

1. **Complexity**: Kubernetes, GitOps, secrets management, observability - each requires
   deep expertise to deploy and maintain correctly.

2. **Cloud Costs**: Public cloud costs escalate quickly, especially for compute-intensive
   workloads. Many organizations have underutilized on-premises hardware.

3. **Compliance**: Some industries require on-premises data processing and cannot use
   public cloud services.

4. **DevOps Scarcity**: Skilled infrastructure engineers are expensive and hard to find.
   Small teams often lack the expertise to build reliable platforms.

---

## Solution

A productized platform delivered through multiple tiers to serve different customer segments:

### Tier 1: CTO Appliance (MVP)
A bootable ISO that installs a complete, production-ready platform on bare-metal servers:

- Boot from USB/ISO
- Follow simple setup wizard (5-10 minutes)
- Platform installs and configures automatically (15-30 minutes)
- Access dashboard and start deploying applications
- **No Kubernetes knowledge required**

### Tier 2: CTO Enterprise (Post-MVP)
Scalable multi-node clusters for growing organizations:

- Start with Appliance, scale to enterprise
- Add nodes via simple enrollment process
- High availability control plane
- Distributed storage across nodes
- Enterprise support and SLAs

### Tier 3: CTO Platform (Post-MVP)
Helm charts and operators for teams with existing Kubernetes:

- Bring your own Kubernetes (any distribution)
- Install CTO platform components only
- Full GitOps integration
- For DevOps-savvy teams
- Lower price point (no infrastructure management)

---

## Target Personas

### Tier 1 (Appliance) Personas

**Primary: Technical Founder / Lead Developer**
- Has development experience but limited infrastructure expertise
- Wants to focus on product, not operations
- Budget-conscious, prefers CapEx over OpEx
- Comfortable with basic server hardware

**Secondary: Small IT Team**
- Manages on-premises infrastructure for a company
- Familiar with basic Linux administration
- Looking to modernize without massive learning curve
- Needs vendor support option

### Tier 2 (Enterprise) Personas

**Primary: Growing Startup / Scale-up**
- Started with Appliance, now needs to scale
- Team has grown, reliability is critical
- Needs HA but still wants managed simplicity
- May have compliance requirements

**Secondary: Enterprise Edge Deployment**
- Deploying standardized infrastructure to multiple locations
- Needs consistent, repeatable deployments
- Values air-gapped capability
- Requires enterprise support and SLAs

### Tier 3 (Platform-Only) Personas

**Primary: DevOps Team with Existing K8s**
- Already running Kubernetes (cloud or on-prem)
- Has infrastructure expertise in-house
- Wants the CTO agent capabilities without re-platforming
- Comfortable with Helm and GitOps

**Secondary: Cloud-Native Consultancy**
- Implementing CTO for clients
- Clients have varied infrastructure
- Needs flexibility in deployment options
- Values documentation and support

- OTA (Over-The-Air) update operator
- Automatic security patches within maintenance windows

#### Platform Guardian Agent (Self-Healing)
The platform includes a dedicated maintenance agent that keeps the environment healthy
without requiring infrastructure expertise from users.

**Continuous Monitoring:**
- Cluster health (nodes, pods, services)
- Certificate expiration (auto-renewal)
- Storage capacity and cleanup
- Resource utilization (CPU, memory, disk)
- Network connectivity
- Backup verification
- Security posture

**Automatic Remediation:**
- Restart failed services
- Clean up orphaned resources
- Rotate expiring certificates
- Scale resources during pressure
- Clear filled logs/temp storage
- Reconnect dropped integrations
- Apply safe security patches

**Proactive Detection:**
- Predict storage exhaustion before it happens
- Identify resource contention patterns
- Detect configuration drift
- Flag deprecated API usage before upgrades
- Monitor for security vulnerabilities

**Phone-Home Reporting (opt-in):**
- Anonymous health telemetry to 5D Labs
- Early warning of common issues
- Aggregate data improves platform for all users
- Critical alerts can trigger proactive support
- No sensitive data transmitted (code, secrets, etc.)

**User Experience:**
- Dashboard shows platform health score
- Plain-English explanations of issues
- "Guardian handled X issues this week" summary
- Escalation to human only when necessary
- No SSH, no kubectl required for routine maintenance

#### Security
- Cryptographic license validation (offline-capable)
- TLS everywhere by default
- Automatic certificate rotation
- Network policies enabled
- Secrets encrypted at rest

## Hardware Requirements

### MVP: Certified Hardware (v1.0)

For MVP, we focus on **one or two certified server configurations** to ensure
a consistent, tested experience. This is the "it just works" path.

**Primary Certified Platform: Dell PowerEdge R660/R760**
- Most popular enterprise server line
- Wide availability (new and refurbished)
- Well-documented, good driver support
- Dell iDRAC for remote management

**Certified Configuration:**
- CPU: Intel Xeon Silver 4410Y (12 cores) or better
- RAM: 64 GB DDR5 ECC (expandable)
- Storage: 2x 960GB NVMe SSD (BOSS-N1 boot + data)
- Network: Broadcom 10GbE dual-port
- Boot: UEFI with Secure Boot
- RAID: PERC H965i in HBA mode (passthrough)

**Alternative Certified Platform: HPE ProLiant DL360 Gen11**
- Similar specs to Dell
- For customers with HPE preference/contracts

### General Requirements (for non-certified hardware)

Users can run on non-certified hardware at their own risk.
Community-reported compatible hardware will be documented.

**Minimum:**
- CPU: 8 cores (x86_64, Intel Xeon or AMD EPYC class)
- RAM: 32 GB ECC
- Storage: 500 GB NVMe SSD
- Network: 1 Gbps Ethernet
- Boot: UEFI preferred, Legacy BIOS supported

**Recommended:**
- CPU: 16+ cores
- RAM: 64+ GB ECC
- Storage: 1+ TB NVMe SSD
- Network: 10 Gbps Ethernet
- Boot: UEFI with Secure Boot

### Hardware Compatibility Roadmap

| Version | Hardware Support |
|---------|------------------|
| v1.0 (MVP) | Dell PowerEdge R660/R760, HPE DL360 Gen11 |
| v1.1 | Supermicro SYS series, Lenovo ThinkSystem |
| v1.2 | Intel NUC (dev/small teams), Mini PCs |
| v1.3 | ARM64 servers (Ampere, AWS Graviton-compatible) |
| v1.4 | GPU servers (NVIDIA A100/H100, RTX 4090) |
| v2.0 | Community-contributed hardware profiles |

### Why Certified Hardware?

1. **Guaranteed Experience**: We test every release on certified hardware
2. **Faster Support**: Issues on certified hardware get priority
3. **Driver Confidence**: No surprises with network/storage drivers
4. **Documentation**: Step-by-step guides specific to your hardware
5. **Refurbished Option**: Dell R650/R750 (previous gen) available cheap

### Storage Notes
- Hardware RAID controllers should be in HBA/passthrough mode
- NVMe drives preferred for performance
- Minimum 20% free space required for upgrades and operations
- BOSS cards (Dell) or M.2 boot devices work well

## Appendix: Competitive Analysis

### Harvester (SUSE)
- Pros: Full HCI, VM support via KubeVirt, open source
- Cons: Opinionated, complex, VM-focused not container-native
rentiation**: Simpler, developer-focused, agent-first
- **Tier 2 differentiation**: Container-native vs VM-centric approach

### Rancher
- Pros: Multi-cluster management, large community
- Cons: Management layer only, doesn't provide the infrastructure
- **Tier 1/2 differentiation**: Complete platform, not just management
- **Tier 3 differentiation**: Agent capabilities beyond basic management

### k3s / k0s
- Pros: Lightweight, easy to install
- Cons: DIY - no platform components included
- **Tier 1 differentiation**: Batteries-included, zero-config to production
- **Tier 3 differentiation**: Full platform stack vs bare Kubernetes

### OpenStack
- Pros: Mature, comprehensive IaaS
- Cons: Complex, heavyweight, requires dedicated team
- **All tiers differentiation**: Simplicity, Kubernetes-native, modern stack

### Talos + Sidero/Omni
- Pros: Same base OS, bare-metal provisioning
- Cons: Infrastructure only, no application platform
- **Tier 1 differentiation**: Full platform stack included
- **Tier 2 differentiation**: Simpler than Sidero, integrated platform

### Portainer / Lens
- Pros: Good UI for K8s management
- Cons: UI layer only, no platform capabilities
- **All tiers differentiation**: AI agents, GitOps-native, full platform

### Coder / Gitpod
- Pros: Developer environments
- Cons: Cloud-focused, no self-hosted bare-metal story
- **Tier 1/2 differentiation**: Own your infrastructure + AI agents
- **Tier 3 differentiation**: Complements rather than competes

### Our Unique Value Proposition
1. **AI Agent-First**: Purpose-built for agentic workflows
2. **True Bare-Metal**: Real on-premises ownership, not cloud dependency
3. **Appliance Simplicity**: Consumer-grade experience for enterprise infrastructure
4. **Growth Path**: Single-node to enterprise without re-platforming
5. **Flexibility**: From USB-install to existing-K8s, meet customers where they are
6. **Built-in SRE**: Guardian Agent maintains the platform automatically
   - No competitor offers self-healing infrastructure for self-hosted
   - Bridges gap between "managed service" reliability and "self-hosted" control
   - Users get reliability without needing infrastructure expertise
   - Phone-home telemetry (opt-in) enables proactive support
