## Codebase Patterns
- Intake CLI JSON parsing: Use `text.contains("```json")` not `text.find()` for code block detection
- Task file structure: `.tasks/tasks/tasks.json` with `metadata` and `tasks` arrays
- Agent hints: Task 1 should always be infrastructure (agentHint='bolt')
- Provider.rs fix commit: 9ee45367d - handles prefill-generated output from Claude
- Linear webhooks: Regular webhooks don't include AgentSession events - use OAuth app webhooks
- Manual trigger: POST /trigger/intake with {"issueId": "CTOPA-XXX"} bypasses webhook dependency
- CodeRuns vs Argo: cto___jobs shows Argo workflows, use `kubectl get coderuns -n cto` for CodeRuns
- In-cluster vs local: Always verify in-cluster image matches local fix before E2E testing
- PM server requires KUBECONFIG environment variable when running locally (default uses ~/.kube/config which may not exist)
- For local dev, set LINEAR_SKIP_SIGNATURE_VERIFICATION=true to bypass webhook signature checks
- PM logs are written to /private/tmp/cto-pm.log when started via nohup
- Once an agent session comment exists on a Linear issue, reassigning delegate won't trigger a new session
- Intake workflow creates ConfigMap in cto namespace before creating CodeRun
