### Analysis Performed
1. Examined recent 15 builds:
   - 21087277467: in_progress (74min+)
   - 21069563331: success (98min)
   - 21058443913: success (318min - severe cache issues)
   - Multiple cancelled builds (48min, 30min, 31min, etc.)

2. Job breakdown from successful run 21069563331:
   - build-runtime: 21min
   - build-sidecar: 5min
   - build-agents (claude): 34min (slowest)
   - build-agents (code): 24min
   - build-agents (factory): 23min
   - build-agents (codex): 21min
   - build-agents (opencode): 19min
   - build-agents (gemini): 18min
   - build-agents (cursor): 9min

3. Runtime image analysis:
   - 61 layers total
   - Largest layer: 531MB (apt packages)
   - Key bloat: Playwright+Chromium (~400MB), security scanners, PHP/Ruby/Perl

4. Cache analysis:
   - GHA cache configured with per-agent scopes
   - Registry cache configured but may not be pulling effectively
   - CACHE_BUST arg defeats caching on every run

### Key Findings
1. **CACHE_BUST arg is counterproductive**: `CACHE_BUST=${{ github.run_id }}` invalidates cache every run
2. **Runtime image too large**: 531MB apt layer + many downloads = slow pulls
3. **Duplicate Rust installations**: Both root and node user have full Rust toolchain
4. **Serial agent builds**: max-parallel: 2 but agents wait on runtime
5. **No pre-warming**: Each agent job pulls runtime from scratch
