## Codebase Patterns
- Use `just mp` to start all local services (PM, Controller, Healer)
- Use `just tunnel` to start Cloudflare tunnel
- Use `just webhook-dev` to point GitHub webhook to dev tunnel
- Use `just preflight` for comprehensive environment check
- Health endpoints: PM=8081, Controller=8080, Healer=8082
- Linear uses OAuth credentials (not API key)
- Agent OAuth tokens synced from K8s secrets to .env.local
- Skills are loaded from `/templates/skills/` based on `skill-mappings.yaml`
- Test strategy is in task-XXX.md files (generated by `intake generate`) under `## Test Strategy` section
- Subtasks enable parallel execution via subagent dispatch
- Healer Play API health endpoint is on port 8083 (not 8082)
- Linear webhook can be queried via GraphQL API: `webhooks { nodes { url enabled } }`
- Intake MCP tool attaches docs as Linear documents (linked in description), not as file attachments
- Intake CodeRuns use format: `intake-<issue-id>-<timestamp>` (e.g., intake-ctopa-807-1768504019)
- Intake CLI generates `## Test Strategy` section in prompt.md when task.test_strategy is non-empty (crates/intake/src/bin/cli.rs:1020-1024)
- AI JSON parsing errors occur when Claude includes explanatory text with JSON output - fix in `extract_json_continuation` (provider.rs) handles this by extracting from `{"id":` pattern
- Intake release pipeline: commit fix → PR to main → merge → tag (v*.*.*) → CI builds binaries → runtime image picks up new binary on rebuild
- Successful intake example: 5dlabs/alerthub-multi-platform-notification-system has complete tasks.json with testStrategy, agentHint, subtasks
- `autoAppendDeployTask` defaults to false; enable via `"defaults": { "intake": { "autoAppendDeployTask": true } }` in cto-config.json
- When testing new CLI commands, reinstall binary: `cargo install --path crates/<crate> --force`
- Linear sync is pull-based via `intake_sync_task` MCP tool (not webhook-driven)
