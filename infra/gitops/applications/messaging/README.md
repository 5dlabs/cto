# Self-Hosted Messaging Infrastructure

This directory contains ArgoCD applications for self-hosted messaging and notification services.

## Components

### Push Notifications: ntfy

**License**: Apache 2.0 / GPL-2.0 (dual licensed) - ✅ Redistributable

[ntfy](https://ntfy.sh/) is a simple HTTP-based pub-sub notification service that replaces Firebase Cloud Messaging (FCM) for self-hosted environments.

**Features**:
- UnifiedPush support for Android (no Google Play Services required)
- Web Push via VAPID keys
- HTTP/WebSocket subscription
- Rate limiting and access control
- Single binary, no dependencies

**Usage in AlertHub**:
- Android: Apps use UnifiedPush with ntfy as the distributor
- Web: Web Push API with VAPID keys generated by ntfy
- iOS: Still requires direct APNs connection (Apple requirement)

**Endpoint**: `https://push.5dlabs.ai` (internal: `ntfy.messaging.svc:80`)

---

### Email Testing: Mailpit

**License**: MIT - ✅ Redistributable

[Mailpit](https://github.com/axllent/mailpit) is an email and SMTP testing tool for development environments.

**Features**:
- SMTP server for capturing emails
- Modern web UI for viewing emails
- REST API for integration testing
- No external dependencies

**Usage**:
- Development: All services send to `mailpit.email.svc:1025`
- Web UI: `http://mailpit.email.svc:8025`

### Production Email Options

For production email delivery, choose based on your requirements:

#### Option 1: Postal (Recommended for full-featured)

**License**: MIT - ✅ Redistributable

[Postal](https://github.com/postalserver/postal) is a complete mail delivery platform.

**Pros**:
- Full-featured MTA
- Delivery tracking, bounce handling
- Web UI for management
- Webhook notifications

**Cons**:
- Requires PostgreSQL and Redis (additional infrastructure)
- More complex to operate

#### Option 2: Simple SMTP Relay (Postfix)

**License**: IBM Public License - ✅ Redistributable

A simple Postfix relay for outbound email only.

**Pros**:
- Lightweight, battle-tested
- Minimal dependencies
- Easy to configure

**Cons**:
- No tracking or management UI
- Basic functionality only

#### Option 3: External SMTP Provider

Configure services to use any SMTP endpoint. Compatible with:
- Self-hosted Postal/Postfix/Maddy
- Cloud providers (if desired): SendGrid, AWS SES, Mailgun

**Note**: For AlertHub, the Integration Service accepts SMTP configuration per-tenant, so different deployments can use different backends.
