# Details:
The Toolman project already includes a complete Helm chart at `toolman/charts/toolman/` with the following features:
- Complete deployment configuration
- ConfigMap for server definitions (toolman-config)
- Service exposure on port 3000
- Support for stdio, SSE, and HTTP transport types
- Pre-configured servers: brave-search, memory, terraform, kubernetes, solana, rustdocs, reddit
- PVC for persistence

Tasks:
1. Review the existing chart structure and configuration
2. Analyze values.yaml for customization options
3. Review pre-configured MCP servers in the ConfigMap
4. Identify any platform-specific customizations needed
5. Test deployment with default values
6. Document any required value overrides for our environment

# Test Strategy:
Run `helm lint toolman/charts/toolman/` to validate the existing chart. Deploy to a test namespace using `helm install toolman-test toolman/charts/toolman/ --dry-run --debug` to verify template rendering. Perform actual deployment to test namespace and verify all resources are created correctly. Check that pre-configured MCP servers are accessible.
